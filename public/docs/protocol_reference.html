1. SENSOR TEENSY → JETSON

TCP 192.168.1.178:23 — newline-delimited ASCII. Parsed in comms.cpp sensorRxLoop callback.

Message Format Fields Rate Notes
SENSOR_READY SENSOR_READY\n — Once Sent on new client connect. Triggers IMU_ON + IMU_RATE from Jetson.
YPR YPR <yaw> <pitch> <roll>\n yaw float degrees
pitch float degrees
roll float degrees		 50 Hz (default) BNO085 body IMU rotation vector. Space-separated. Also forwarded to slam_bridge as imu_data JSON.
$GNGGA $GNGGA,hhmmss.ss,lat,N/S,lon,E/W,quality,sats,hdop,alt,M,...*XX\n Standard NMEA GGA fields. Validated via XOR checksum. 1 Hz BN880 GPS fix. Also accepts $GPGGA. Parsed for lat/lon/alt/quality/sats. Forwarded to slam_bridge as gps_data JSON.
$GNRMC $GNRMC,hhmmss.ss,A/V,lat,N/S,lon,E/W,speed,course,...*XX\n speed knots, course degrees true 1 Hz Also accepts $GPRMC. Validated via checksum. Only processed if status=A (active). Forwarded to slam_bridge as gps_rmc JSON.
ENC ENC <left_sps> <right_sps>\n left_sps int steps/sec
right_sps int steps/sec		 10 Hz Wheel encoder speeds. Space-separated integers.
TOF TOF <dist_m>\n dist_m float metres ~10 Hz Time-of-flight distance sensor. 16-byte binary frame from sensor, converted to metres by Teensy.

2. JETSON → SENSOR TEENSY

TCP 192.168.1.178:23 — newline-delimited ASCII commands.

Command Format Notes
IMU_ON IMU_ON\n Enable YPR streaming from BNO085. Sent automatically on connect.
IMU_OFF IMU_OFF\n Disable YPR streaming.
IMU_RATE IMU_RATE <hz>\n hz int 1–200. Set YPR output rate. Default: 50. Sent automatically on connect.

3. CONTROL TEENSY → JETSON

TCP 192.168.1.177:23 — newline-delimited ASCII. Parsed in comms.cpp controlRxLoop callback.

Message Format Fields Notes
ENC ENC <left> <right>\n left int steps/sec
right int steps/sec		 Wheel encoder feedback. Stored in SensorStore encoders.
PTU_YPR PTU_YPR <yaw> <pitch> <roll>\n yaw float degrees
pitch float degrees
roll float degrees		 PTU-mounted BNO085 IMU orientation. Currently reverted from deployed binary pending FPS fix.

4. JETSON → CONTROL TEENSY

TCP 192.168.1.177:23 — newline-delimited ASCII commands.

Command Format Fields Notes
LS..RS.. LS<left>RS<right>\n left int steps/sec
right int steps/sec		 Set wheel motor speeds. Example: LS400RS-400 (turn right). Negative = reverse.
VP..T.. VP<pan_sps>T<tilt_sps>\n pan_sps int steps/sec
tilt_sps int steps/sec		 PTU velocity mode. Example: VP200T0 (pan right). 0 = stop axis.
P..T.. P<pan>T<tilt>\n pan int position (steps)
tilt int position (steps)		 PTU absolute position mode. P0T0 = centre. Used by "manual_ptu:centre" command.

5. JETSON → VPS — STATUS JSON

TCP port 9999, newline-delimited JSON. Pushed every 1 second (configurable via status_push_interval_ms) and after each command response.

FULL STATUS JSON SCHEMA

{
  "imu": {
    "yaw": float,        // degrees
    "pitch": float,      // degrees
    "roll": float,       // degrees
    "age_ms": float,     // ms since last reading
    "stale": bool        // true if age_ms > telemetry_stale_ms
  },
  "gps": {
    "lat": double,       // decimal degrees
    "lon": double,       // decimal degrees
    "alt": float,        // metres above MSL
    "speed_knots": float,
    "course_deg": float, // true heading
    "quality": int,      // GGA fix quality (0=none, 1=GPS, 2=DGPS)
    "sats": int,         // satellites in use
    "age_ms": float,
    "stale": bool
  },
  "encoders": {
    "left": int,         // steps/sec
    "right": int,        // steps/sec
    "age_ms": float,
    "stale": bool
  },
  "distance_m": float,         // TOF distance in metres
  "distance_age_ms": float,
  "lidar_fwd_m": float,        // closest forward lidar reading (metres)
  "lidar_fwd_age_ms": float,
  "lidar_obstacle": bool,      // true if obstacle within 0.5m forward
  "slam_pose": {
    "x": float,           // SLAM map frame X
    "y": float,           // SLAM map frame Y
    "theta": float,       // heading radians
    "valid": bool
  },
  "detection_fps": float,      // YOLOv8 inference FPS
  "stream_quality": int,       // JPEG quality 1-100
  "target_person_id": int,     // active follow target (-1 = none)
  "sensors": {                  // generic sensor slots (auto-populated)
    "<key>": {
      "value": float,
      "age_ms": float,
      "stale": bool
    }
    // Examples: cpu_temp, gpu_temp, cpu_pct, gpu_pct, ram_pct
  },
  "mode": string,             // "follow" | "manual" | "mission" | "stopped"
  "mission": {
    "running": bool,
    "waypoint_idx": int,    // current waypoint index (0-based)
    "fault": string,        // "" | "gps_lost" | "stuck" | "wheel_fail"
    "id": string,           // mission ID (if loaded)
    "name": string,         // mission display name
    "waypoint_count": int,  // total waypoints
    "target_lat": double,   // current target waypoint lat
    "target_lon": double    // current target waypoint lon
  }
}

6. VPS → JETSON — WEB COMMANDS

TCP port 9999, newline-delimited plain text. Handled by comms.cpp handleWebCommand(). Response is status JSON or custom JSON, one line per command.

Command Format Action
MODE CONTROL
stop_follow stop_follow\n Transition to STOPPED mode. Zeros all actuators.
emergency_stop emergency_stop\n Same as stop_follow. Transition to STOPPED mode.
resume_follow resume_follow\n Transition to FOLLOW mode. Activates person-tracking PID.
MANUAL CONTROL
manual_wheel manual_wheel:<L>:<R>\n Set manual wheel speeds. L, R = int steps/sec. Auto-enters MANUAL mode.
manual_ptu (velocity) manual_ptu:<P>:<T>\n Set PTU velocity. P, T = float steps/sec.
manual_ptu (centre) manual_ptu:centre\n Stop PTU velocity and move to centre position (P0T0).
MISSION COMMANDS
mission_save mission_save:<json>\n Save mission to disk without starting. JSON must include id field.
mission_start mission_start:<json>\n Save and start a mission. Transitions to MISSION mode.
mission_abort mission_abort\n Abort running mission. Transitions to STOPPED mode.
mission_load mission_load:<id>\n Load saved mission by ID from disk and start it.
list_missions list_missions\n Returns JSON array of saved missions (custom response).
mission_skip_wp mission_skip_wp\n Skip current waypoint, advance to next.
mission_resume mission_resume:<id>\n Resume mission from last saved waypoint.
TARGETING / LEARN
set_target set_target:<person_id>\n Set active follow target by gallery person ID. person_id = int.
learn_start learn_start:<person_id>\n Begin collecting appearance crops for AI description.
learn_stop learn_stop\n Stop crop collection, trigger Claude API description.
CAMERA / STREAM
stereo_start stereo_start\n Start stereo depth camera (ZMQ 5557 + 5558).
stereo_stop stereo_stop\n Stop stereo depth camera.
stream_quality stream_quality:<q>\n Set main MJPEG stream JPEG quality. q = int 1–100.
depth_quality depth_quality:<q>\n Set depth disparity JPEG quality. q = int 1–100.

7. JETSON → VPS — EVENT LINES

TCP port 9999, newline-delimited JSON. Sent via sendWebEvent() — server.js detects {"type":"event"} and re-broadcasts without overwriting lastStatus cache.

Event JSON Format Origin
mission_completed {"type":"event","event":"mission_completed","id":"...","name":"..."} mission.cpp — natural mission completion
mission_faulted {"type":"event","event":"mission_faulted","id":"...","name":"...","fault":"gps_lost|stuck|wheel_fail"} mission.cpp — mission failure
new_target {"type":"event","event":"new_target","target_id":<int>} object_detection.cpp — new person added to gallery
learn_progress {"type":"event","event":"learn_progress","target_id":<int>,"count":<int>,"total":<int>} object_detection.cpp — crop collection progress
learn_crops {"type":"learn_crops","target_id":<int>,"crops":["<base64_jpg>",...]} object_detection.cpp — collected crops sent to VPS for Claude API
scan {"type":"scan","obs":<bool>,"fwd":<float>,"pts":[[<angle_deg>,<dist_m>],...]} comms.cpp broadcastScan() — lidar scan data for web radar
slam_map {"type":"slam_map","png":"<base64>","res":<float>,"ox":<float>,"oy":<float>,"w":<int>,"h":<int>} slam_bridge.py via comms.cpp — occupancy grid PNG, forwarded verbatim

8. WEBSOCKET BROWSER ↔ VPS

WebSocket on port 3000, path /ws?tok=<token>. Token obtained via GET /api/ws-token (valid 30s, one-time use). JSON messages both directions.

BROWSER → VPS (COMMANDS)

Format Action
{"cmd":"<command_string>"} Forwarded verbatim to Jetson via TCP 9999. All commands from section 6 are valid. Returns {"type":"ack","cmd":"...","ok":true|false}.

VPS → BROWSER (EVENTS)

type JSON Fields Notes
connection {"type":"connection","status":"connected"|"disconnected"} Sent on WS open (current state) and on Jetson connect/disconnect.
status {"type":"status","data":<status_json | null>} Full telemetry from Jetson (section 5). Sent on connect (cached) and every push. data: null sent on Jetson disconnect.
ack {"type":"ack","cmd":"...","ok":true|false} Sent to the requesting client after each command.
mission_status {"type":"mission_status","event":"...","missionId":"...","name":"...","fault":"..."} Events: executing, completed, aborted, mission_faulted, abort_pending, pushed.
new_target {"type":"new_target","target_id":<int>} Triggers gallery refresh in browser.
learn_progress {"type":"learn_progress","target_id":<int>,"count":<int>,"total":<int>} Crop collection progress during learn mode.
learn_status {"type":"learn_status","target_id":<int>,"status":"analysing"|"error","count":<int>,"message":"..."} Claude API processing status. Generated by server.js.
target_description {"type":"target_description","target_id":<int>,"description":"..."} AI-generated appearance description. Sent on connect for all known targets, and on new description.
scan {"type":"scan","obs":<bool>,"fwd":<float>,"pts":[[<deg>,<m>],...]} Lidar scan for radar display. Forwarded from Jetson event.
slam_map {"type":"slam_map","png":"...","res":...,"ox":...,"oy":...,"w":...,"h":...} Occupancy grid. Cached — sent on connect.

9. SLAM BRIDGE PROTOCOL

TCP port 9997, bidirectional newline-delimited JSON between C++ app and slam_bridge.py. Auto-reconnect with 2s retry.

C++ APP → SLAM BRIDGE

type JSON Format Published As Rate
imu_data {"type":"imu_data","yaw":<f>,"pitch":<f>,"roll":<f>} ROS2 /imu/data (sensor_msgs/Imu) 50 Hz
gps_data {"type":"gps_data","lat":<f>,"lon":<f>,"alt":<f>,"quality":<i>,"sats":<i>} ROS2 /gps/fix (sensor_msgs/NavSatFix) 1 Hz
gps_rmc {"type":"gps_rmc","speed_knots":<f>,"course_deg":<f>} Internal heading calibration 1 Hz
odom_data {"type":"odom_data","left":<i>,"right":<i>,"age_ms":<f>} ROS2 /odom (nav_msgs/Odometry) 10 Hz
plan_path {"type":"plan_path","from_lat":<f>,"from_lon":<f>,"to_lat":<f>,"to_lon":<f>} Nav2 ComputePathToPose action On demand

SLAM BRIDGE → C++ APP

type JSON Format Notes
scan_full {"type":"scan_full","obs":<bool>,"fwd":<float>,"pts":[[<deg>,<m>],...]} obs = obstacle within 0.5m fwd. fwd = closest forward distance. pts = polar points [angle_deg, dist_m]. Stored in SensorStore, broadcast to web as scan event.
slam_pose {"type":"slam_pose","x":<float>,"y":<float>,"theta":<float>} SLAM Toolbox estimated pose in map frame. theta in radians.
slam_map {"type":"slam_map","png":"<base64>","res":<float>,"ox":<float>,"oy":<float>,"w":<int>,"h":<int>} Occupancy grid as base64 PNG. res = m/pixel, ox/oy = origin, w/h = dimensions. Forwarded to web verbatim.
planned_path (success) {"type":"planned_path","ok":true,"waypoints":[[<lat>,<lon>],...]} Nav2-planned path as GPS sub-waypoints (~1m spacing). Routed to Nav2Planner.
planned_path (failure) {"type":"planned_path","ok":false,"error":"..."} Errors: "Nav2 not available", "No GPS reference yet", "Nav2 planner server not ready", "GPS→map conversion failed", "No valid path found".
ODOMETRY STATE MACHINE

slam_bridge.py tracks encoder freshness: ODOM_INACTIVE → ODOM_ACTIVE after 5 consecutive fresh encoder reads. ODOM_ACTIVE → ODOM_INACTIVE after 3 consecutive stale reads. On each transition: saves SLAM map, kills SLAM Toolbox, restarts with matching params YAML (with/without odom).

10. ZMQ PUB/SUB PORTS

All ZMQ sockets use PUB/SUB pattern on localhost (127.0.0.1). Published by C++ app, consumed by Python bridges.

Port Publisher Subscriber Content Notes
5555 object_detection.cpp mjpeg_bridge.py Detection-annotated JPEG frames Main camera stream. YOLOv8 bounding boxes + tracking overlaid. CONFLATE=1 (latest only).
5557 stereo_depth.cpp (Future: stereo_ros2_bridge / RTAB-Map) Stereo rectified frame pairs Left+right rectified images for 3D SLAM. Not yet consumed.
5558 stereo_depth.cpp mjpeg_bridge.py Colourised disparity JPEG Depth map visualization. Fed to mjpeg_bridge /depth_stream endpoint.

11. MJPEG BRIDGE — HTTP ENDPOINTS

Python HTTP server on Jetson port 8080. Serves MJPEG streams from ZMQ and target gallery from disk.

Method Path Response Notes
GET /stream multipart/x-mixed-replace MJPEG Main detection camera stream. 25 fps target. Boundary: --tankframe. From ZMQ 5555.
GET /depth_stream multipart/x-mixed-replace MJPEG Stereo depth disparity stream. From ZMQ 5558.
GET /depth_snapshot image/jpeg Single depth frame JPEG. 503 if no frame available.
GET /targets application/json JSON array: [{"id":int,"name":"...","thumb_url":"/targets/<id>/thumb","embed_count":int},...]
GET /targets/<id>/thumb image/jpeg Person thumbnail from gallery. 404 if not found.
POST /targets/<id>/name application/json Rename person. Body: {"name":"..."}. Writes to /home/james/tank_targets/<id>/name.txt.

12. VPS HTTP API ENDPOINTS

Node.js server on VPS port 3000. Session-authenticated (7-day cookie). Proxies MJPEG + gallery from Jetson.

Method Path Auth Response / Action
AUTHENTICATION
GET /login No Login page HTML.
POST /login No Body: password=.... Sets session cookie on success. Redirects to / or /login?err=1.
POST /logout Yes Destroys session, redirects to /login.
GET /api/ws-token Yes {"token":"..."} — one-time WS auth token, valid 30 seconds.
VIDEO STREAMS (proxied from Jetson :8080)
GET /stream Yes MJPEG relay. Fan-out: single upstream to all viewers. 503 if offline.
GET /snapshot Yes Single JPEG (for Safari/iOS). 503 if no frame.
GET /depth_stream Yes Depth MJPEG relay. 503 if offline.
GET /depth_snapshot Yes Single depth JPEG. 503 if no frame.
TARGET GALLERY (proxied from Jetson :8080)
GET /api/targets Yes JSON array of known persons.
GET /api/targets/:id/thumb Yes Person thumbnail JPEG.
PUT /api/targets/:id/name Yes Rename person. Body: {"name":"..."}.
MISSION API
GET /api/missions Yes JSON list: [{"id","name","type","created","updated","waypointCount"},...]
GET /api/missions/state Yes Current mission state from Jetson: {"mission_id","waypoint_idx"} or {}.
GET /api/missions/:id Yes Full mission JSON (waypoints, metadata).
POST /api/missions Yes Create/save mission. Body: mission JSON. Returns {"ok":true,"id":"..."}.
PUT /api/missions/:id Yes Update mission. Body: mission JSON.
DELETE /api/missions/:id Yes Delete mission from VPS.
POST /api/missions/:id/push Yes Push mission to Jetson (save only, no execute). Sends mission_save: to TCP 9999. Optional body: {"waypoints":[...]} for route-expanded points.
POST /api/missions/:id/execute Yes Execute mission on Jetson. Sends mission_start: to TCP 9999. Optional body: {"waypoints":[...]}.
POST /api/missions/abort Yes Abort mission. Sends mission_abort to TCP 9999. Queues if offline. Returns {"ok":bool,"queued":bool}.
ROUTING
POST /api/route Yes OSRM road routing. Body: {"points":[{"lat","lng"},...],"profile":"foot"|"car"}. Returns {"points":[...],"distance","duration"}.
