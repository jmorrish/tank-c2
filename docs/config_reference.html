1. CONFIG LOADING ORDER

Precedence (highest wins):

1. CLI arguments -- --ptu-ip, --sensor-ip, --detect-cam, etc.
2. config.json -- JSON file loaded at startup (default: ./config.json in working dir)
3. Compiled defaults -- values in runtime_config.h struct initializers

The config file path itself can be overridden: --config /path/to/config.json
Missing keys in config.json are silently ignored (compiled default used).
Type mismatches are silently ignored (compiled default used).
If the config file is missing or unparseable, all compiled defaults are used and a warning is logged.

2. CLI ARGUMENTS

Usage: ./yolov8 <engine_path> [options]

Argument Value Default Description
<engine_path> string (positional, required) -- Path to YOLOv8 TensorRT engine file (e.g. yolov8n.engine)
--config <path> string config.json Path to runtime config JSON file. Pre-scanned before other args.
--headless flag false Disable OpenCV GUI windows (required for SSH / no display)
--auto-continue flag false Skip interactive "Press Enter" prompts at startup
--no-hw flag false Skip all hardware connections (Teensy control + sensor). Web IPC still starts.
--ptu-ip <ip:port> string from config teensy_ip:teensy_port Override Control Teensy IP and port (e.g. 192.168.1.177:23)
--ip <ip> string from config teensy_ip Override Control Teensy IP only
--port <n> int from config teensy_port Override Control Teensy port only
--sensor-ip <ip:port> string from config sensor_ip:sensor_port Override Sensor Teensy IP and port. If no colon, sets IP only.
--sensor-port <n> int from config sensor_port Override Sensor Teensy port only
--detect-cam <dev> string auto-detect Detection camera device (e.g. /dev/video0). Empty = runtime scan.
--stereo-cam <dev> string auto-detect Stereo camera device (identified by 2560x720 capability). Empty = disabled.
Example:
./yolov8 /home/james/YOLOv8-TensorRT/yolov8n.engine --auto-continue --headless --config /home/james/config.json

4. SENSOR STALENESS

Field JSON Key Type Default Range Restart? Description
sensor_stale_ms "sensor_stale_ms" double 500.0 > 0 runtime Milliseconds after which sensor readings are considered stale. Affects IMU/GPS/TOF freshness checks.

5. PTU PID GAINS

PID controller for pan/tilt unit tracking. Controls stepper speed based on pixel error from frame center.

Field JSON Key Type Default Range Restart? Description
kp "kp" float 1.5 ≥ 0 runtime Proportional gain. Higher = more aggressive tracking. Too high causes oscillation.
ki "ki" float 0.1 ≥ 0 runtime Integral gain. Eliminates steady-state error. Clamped by max_i.
kd "kd" float 0.1 ≥ 0 runtime Derivative gain. Dampens oscillation. Sensitive to noise.
max_i "max_i" float 1000.0 > 0 runtime Integral windup clamp (absolute). Prevents runaway accumulation.

6. PTU MOTION LIMITS

Velocity and deadband limits for PTU stepper commands sent to Control Teensy.

Field JSON Key Type Default Range Restart? Description
center_threshold "center_threshold" float 20.0 ≥ 0 px runtime Pixel deadband. PTU stops moving when target is within this many pixels of frame center.
min_speed "min_speed" float 0.0 ≥ 0 sps runtime Minimum PTU stepper speed (steps/second). Below this, PTU stops.
max_speed "max_speed" float 2000.0 > 0 sps runtime Maximum PTU stepper speed (steps/second). Clamps PID output.
velo_eps "velo_eps" float 25.0 ≥ 0 sps runtime Velocity change threshold. New command only sent if delta exceeds this (reduces Teensy traffic).
max_delta_velo "max_delta_velo" float 500.0 > 0 sps runtime Max velocity change per step. Rate-limits PTU acceleration to prevent mechanical jerk.

7. PTU SOFT LIMITS

Accumulated step limits from startup position. Prevents PTU from over-rotating and hitting hard stops.

Field JSON Key Type Default Range Restart? Description
pan_limit_steps "pan_limit_steps" float 50000.0 > 0 runtime Max accumulated pan steps (+/-) from startup zero. At 1600 steps/rev = ~31 full revolutions.
tilt_limit_steps "tilt_limit_steps" float 20000.0 > 0 runtime Max accumulated tilt steps (+/-) from startup zero. At 1600 steps/rev = ~12.5 full revolutions.

8. DETECTION

Field JSON Key Type Default Range Restart? Description
confidence_threshold "confidence_threshold" float 0.40 0.0 - 1.0 runtime YOLOv8 detection confidence threshold. Detections below this are discarded. Lower = more detections but more false positives.

9. PERSON RE-ID / GALLERY

BoTSORT tracking with cosine-similarity re-identification against persistent person gallery.

Field JSON Key Type Default Range Restart? Description
gallery_match_thresh "gallery_match_thresh" float 0.75 0.0 - 1.0 runtime Cosine similarity threshold for re-ID match. Higher = stricter matching. Below this, person is treated as unknown.
embed_ema_alpha "embed_ema_alpha" float 0.1 0.0 - 1.0 runtime EMA weight for new embedding updates. Lower = more stable identity (slower adaptation). Higher = faster adaptation to appearance changes.
max_extra_embeddings "max_extra_embeddings" int 100 ≥ 0 runtime Max extra embeddings stored per person in gallery. Caps memory usage for long sessions.

10. AUTO-LEARN

Automatic gallery enrollment of tracked persons that meet quality criteria.

Field JSON Key Type Default Range Restart? Description
auto_learn_interval_ms "auto_learn_interval_ms" int 3000 > 0 ms runtime Minimum interval between auto-learn captures for the same person.
auto_learn_min_conf "auto_learn_min_conf" float 0.80 0.0 - 1.0 runtime Minimum detection confidence for auto-learn capture. Higher than detection threshold to ensure quality.
auto_learn_min_area "auto_learn_min_area" int 8192 > 0 px^2 runtime Minimum bounding box area (pixels) for auto-learn. ~64x128 px. Rejects distant / small detections.
auto_learn_batch_size "auto_learn_batch_size" int 5 ≥ 1 runtime Number of auto-learn samples collected before committing to gallery.

11. LEARN MODE

Manual learn mode quality filters. Applied when operator triggers "learn" for a specific person.

Field JSON Key Type Default Range Restart? Description
learn_sample_ms "learn_sample_ms" int 500 > 0 ms runtime Minimum interval between learn-mode frame captures.
learn_min_crop_px "learn_min_crop_px" int 64 > 0 px runtime Minimum crop dimension (width or height) for learn captures. Rejects tiny crops.
learn_blur_thresh "learn_blur_thresh" float 80.0 > 0 runtime Laplacian variance threshold for blur detection. Below this, image is considered too blurry. Lower = accept blurrier images.
learn_dup_thresh "learn_dup_thresh" float 0.95 0.0 - 1.0 runtime Cosine similarity threshold to reject duplicate embeddings during learn. Above this, new sample is too similar to existing.

12. STREAMING

MJPEG stream and thumbnail encoding parameters.

Field JSON Key Type Default Range Restart? Description
stream_max_w "stream_max_w" int 1280 > 0 px runtime Maximum width for MJPEG stream frames. Frames wider than this are downscaled.
thumb_jpeg_quality "thumb_jpeg_quality" int 85 1-100 runtime JPEG quality for person gallery thumbnails. Higher = better quality, larger files.

13. WHEEL FOLLOW BEHAVIOUR

FollowBehavior: PID-based person tracking using wheels. Activated when operator clicks "follow" on a person.

Field JSON Key Type Default Range Restart? Description
follow_distance_m "follow_distance_m" float 0.5 > 0 m runtime Target follow distance in metres. Robot tries to maintain this distance from tracked person (uses stereo depth).
wheel_gain_distance "wheel_gain_distance" float 20000.0 > 0 runtime Proportional gain for distance error to wheel speed. Higher = more aggressive approach/retreat.
wheel_gain_steer "wheel_gain_steer" float 0.2 ≥ 0 runtime Steering gain. Fraction of base speed applied as differential to left/right wheels for turning.
wheel_max_sps "wheel_max_sps" int 15000 > 0 sps runtime Maximum wheel stepper speed (steps/second). Hard clamp on wheel commands.
max_follow_distance_m "max_follow_distance_m" float 10.0 > follow_distance_m runtime Beyond this distance, wheels stop (person too far to follow). Prevents runaway chasing.

14. WHEEL ENCODER CLOSED-LOOP PI

PI controller for wheel speed regulation using encoder feedback. Not active until wheel encoders are physically wired.

Field JSON Key Type Default Range Restart? Description
wheel_kp "wheel_kp" float 0.3 ≥ 0 runtime Proportional gain for wheel speed PI controller.
wheel_ki "wheel_ki" float 0.05 ≥ 0 runtime Integral gain for wheel speed PI controller.
wheel_pi_max_i "wheel_pi_max_i" float 5000.0 > 0 runtime Integral windup clamp for wheel PI controller.

15. IMU TILT FEEDFORWARD

Field JSON Key Type Default Range Restart? Description
pixels_per_degree "pixels_per_degree" float 5.0 > 0 runtime Camera pixels per degree of IMU tilt. Used for feedforward PTU compensation when chassis tilts.

16. STEREO DEPTH

CUDA StereoBM stereo depth computation. Publishes via ZMQ to ROS2 (port 5557) and mjpeg_bridge (port 5558).

Field JSON Key Type Default Range Restart? Description
stereo_max_fps "stereo_max_fps" float 10.0 > 0 runtime Maximum stereo depth computation rate (fps). Limits GPU load.
stereo_left_fps "stereo_left_fps" float 10.0 > 0 runtime Rate at which left rectified frames are published for RTAB-Map.
stereo_num_disparities "stereo_num_disparities" int 128 16, 32, 48, ... (multiple of 16) restart StereoBM disparity search range. Higher = detects closer objects but slower. Must be multiple of 16.
stereo_block_size "stereo_block_size" int 9 5, 7, 9, ... (odd, ≥ 5) restart StereoBM matching block size. Larger = smoother but less detail. Must be odd and ≥ 5.
stereo_blur_ksize "stereo_blur_ksize" int 3 1, 3, 5, ... (odd) runtime Gaussian blur kernel size applied before stereo matching. 1 = no blur. Must be odd.
stereo_jpeg_quality "stereo_jpeg_quality" int 60 1-100 runtime JPEG quality for colourised disparity frames published to ZMQ 5558.
stereo_calib_xml "stereo_calib_xml" string "/home/james/stereo_calib/stereo_params_cuda.xml" valid file path restart Path to stereo calibration XML (contains map1x/y, map2x/y, P1, T matrices).

17. MISSION NAVIGATION

MissionBehavior: GPS waypoint navigation with Nav2 path planning and obstacle avoidance.

Field JSON Key Type Default Range Restart? Description
mission_gps_loss_abort_s "mission_gps_loss_abort_s" float 30.0 > 0 s runtime Seconds without GPS fix before mission auto-aborts. Safety limit.
mission_stuck_check_s "mission_stuck_check_s" float 30.0 > 0 s runtime Interval for stuck detection check. If robot hasn't moved enough in this window, it's stuck.
mission_stuck_min_move_m "mission_stuck_min_move_m" float 0.5 > 0 m runtime Minimum distance robot must move within stuck_check_s to not be considered stuck.
mission_steer_gain "mission_steer_gain" float 50.0 > 0 runtime SPS per degree of heading error during mission navigation. Higher = sharper turns.
mission_max_steer_frac "mission_max_steer_frac" float 0.6 0.0 - 1.0 runtime Maximum fraction of base speed applied as differential steering. 0.6 = one wheel can be 60% slower.
mission_moving_knots "mission_moving_knots" float 0.5 ≥ 0 kt runtime GPS speed threshold (knots) for considering robot "moving". Below this, GPS course is unreliable for heading.
mission_heading_filter "mission_heading_filter" float 0.05 0.0 - 1.0 runtime Low-pass EMA weight for GPS heading calibration. Lower = more stable but slower to adapt. Compares GPS course to IMU yaw.
mission_ptu_ms_per_deg "mission_ptu_ms_per_deg" float 10.0 > 0 runtime Milliseconds per degree for PTU slew during mission (e.g. centering PTU forward at waypoint).
mission_return_speed_frac "mission_return_speed_frac" float 0.33 0.0 - 1.0 runtime Speed as fraction of max when PTU is returning to center during mission. Slower = gentler return.

18. TELEMETRY

Field JSON Key Type Default Range Restart? Description
telemetry_stale_ms "telemetry_stale_ms" float 2000.0 > 0 ms runtime Telemetry data staleness threshold. Readings older than this are omitted from status JSON.
status_push_interval_ms "status_push_interval_ms" int 1000 > 0 ms runtime Interval between status JSON pushes to VPS via web IPC (port 9999). Lower = more responsive UI but more bandwidth.

19. ZMQ PORTS

ZeroMQ PUB sockets for inter-process communication on the Jetson.

Field JSON Key Type Default Range Restart? Description
zmq_detection_port "zmq_detection_port" int 5555 1-65535 restart ZMQ PUB port for detection data.
zmq_stereo_port "zmq_stereo_port" int 5557 1-65535 restart ZMQ PUB port for stereo rectified frame pairs (subscribed by ROS2/RTAB-Map).
zmq_disparity_port "zmq_disparity_port" int 5558 1-65535 restart ZMQ PUB port for colourised disparity JPEG frames (subscribed by mjpeg_bridge.py).

20. LIDAR

YDLIDAR X3 configuration. Lidar data is forwarded to slam_bridge.py for ROS2 SLAM.

Field JSON Key Type Default Range Restart? Description
lidar_port "lidar_port" string "/dev/ttyUSB0" valid serial device restart Serial port for YDLIDAR X3.
lidar_baud "lidar_baud" int 115200 standard baud rates restart Lidar serial baud rate. X3 uses 115200.
lidar_fwd_arc_deg "lidar_fwd_arc_deg" float 30.0 0 - 180 deg runtime Forward arc half-angle for obstacle detection. 30 = checks +/-30 degrees from front center (60 deg total).
lidar_obstacle_m "lidar_obstacle_m" float 0.5 > 0 m runtime Distance threshold for obstacle detection in forward arc. Objects closer than this trigger avoidance.
lidar_max_range_m "lidar_max_range_m" float 8.0 > 0 m runtime Maximum valid lidar range. Readings beyond this are discarded (X3 max is ~8-12m).
lidar_scan_pts "lidar_scan_pts" int 180 > 0 runtime Number of points per lidar scan forwarded to slam_bridge. Controls angular resolution.

21. PATHS

File system paths on the Jetson. All require restart.

Field JSON Key Type Default Restart? Description
botsort_base "botsort_base" string "/home/james/YOLOv8-TensorRT/BoTSORT-cpp" restart Root directory for BoTSORT tracker. Config, model, and asset paths are derived from this.
missions_dir "missions_dir" string "/home/james/tank_missions" restart Directory for saved mission JSON files (GPS waypoint lists).
targets_dir "targets_dir" string "/home/james/tank_targets" restart Directory for person gallery data. Each person gets a subdirectory with thumbnails + embeddings.
log_path "log_path" string "/tmp/robot_run.csv" restart Path for CSV telemetry log file written during operation.

22. DERIVED PATHS (computed from botsort_base)

These are not in config.json -- they are computed at runtime from the botsort_base field.

Method Computed Path Description
botsortTrackerCfg() {botsort_base}/config/tracker.ini BoTSORT tracker configuration
botsortGmcCfg() {botsort_base}/config/gmc.ini Global motion compensation config
botsortReidModel() {botsort_base}/assets/mobilenetv2_x1_4_msmt17.onnx Re-ID ONNX model for person appearance embedding
botsortReidCfg() {botsort_base}/config/reid.ini Re-ID model configuration
