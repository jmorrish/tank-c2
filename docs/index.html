TANK C2

AUTONOMOUS GROUND VEHICLE COMMAND & CONTROL PLATFORM
Jetson Orin Super · YOLOv8 · ROS2 Humble · Nav2 · SLAM Toolbox

SYSTEM OVERVIEW

Tank C2 is an autonomous tracked vehicle platform built around an NVIDIA Jetson Orin Super. It uses YOLOv8 with TensorRT for real-time object detection, BoTSORT with cosine-similarity re-identification for persistent person tracking, and GPS waypoint navigation with obstacle-aware path planning via ROS2 Nav2 and SLAM Toolbox. A 360-degree YDLIDAR provides SLAM-based mapping and obstacle avoidance. The entire system is monitored and controlled remotely through a mobile-friendly dark web UI hosted on a VPS, connected via WebSocket relay. Stereo depth sensing and FLIR thermal imaging provide additional situational awareness.

HARDWARE STACK

JETSON ORIN SUPER
192.168.1.167
Central brain. Runs C++ detection app (YOLOv8 TensorRT), stereo depth, SLAM, Nav2 path planning, behavior state machine, and all comms.
VPS
87.106.188.134
Cloud relay. Node.js web server hosts the control UI and proxies WebSocket traffic between the browser and Jetson.
CONTROL TEENSY 4.1
192.168.1.177:23
Motor controller. Drives left/right tracks and pan-tilt unit (PTU) via stepper motors. Receives wheel + PTU commands over TCP.
SENSOR TEENSY 4.1
192.168.1.178:23
Sensor hub. BNO085 9-DOF IMU, BN880 GPS, TOF distance sensor, and wheel encoders. Streams data to Jetson over TCP.
FLIR TAU THERMAL
192.168.0.71 (RTSP)
Thermal imaging camera. RTSP stream grabbed by the Jetson for night/low-light detection and situational awareness.

FEATURE ROADMAP

SAFETY-CRITICAL

  • Watchdog failsafe -- Control Teensy auto-stops motors after 500ms no heartbeat
  • Geofence -- Virtual boundary checked in navigateTo(), abort on breach
  • Connection-loss behavior -- Configurable policy: continue / pause / return home

COMMS UPGRADE

  • 4G/5G modem -- Primary internet link replacing WiFi for field use
  • LoRa backup channel -- Emergency stop, GPS beacon, heartbeat (2-15 km range)
  • LoRa base station -- Handheld ESP32 controller with display and e-stop

OPERATIONAL FEATURES

  • Return to home -- Auto-RTH on low battery, connection loss, or operator command
  • Battery monitoring -- Voltage divider to Sensor Teensy ADC, low-voltage alerts
  • Telemetry logging -- GPS track, sensor readings, mode transitions to disk
  • Video recording -- Save detection frames during missions for post-mission replay
  • OTA config update -- Live PID gains, speed limits, obstacle threshold tuning via web UI

DETECTION & BEHAVIOUR

  • Multi-person queue -- Auto-switch to next known person when target lost
  • Night/low-light mode -- Switch to FLIR thermal when visible-light confidence drops
  • Person heatmap -- GPS-tagged sightings overlaid on mission map

ADDITIONAL SENSORS

  • BME680 -- Temperature, humidity, pressure, air quality index via I2C
  • MR60BHA2 60 GHz mmWave -- Vital signs (breathing + heart rate) on stationary targets
  • MQ-2 / MQ-7 gas sensors -- LPG, propane, hydrogen, carbon monoxide detection
  • Voice localisation mic array -- 4x INMP441 MEMS mics, GCC-PHAT TDOA bearing
